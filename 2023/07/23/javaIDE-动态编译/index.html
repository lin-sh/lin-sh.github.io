    java在线IDE-动态编译

    发表于 2023-07-23 分类于 研究生 ， Java Spring

    在JDK 1.6及以后的版本中，引入了javax.tools包，其中包含了用于在运行时进行动态编译的相关类和接口。通过这些API，开发人员可以在程序运行过程中动态地生成Java源代码，然后通过编译器API将其编译成字节码，最后通过类加载器将生成的类加载到JVM中执行。

    在没有动态编译技术之前，要在运行过程中编译Java源代码，通常需要经历以下步骤：

    1. 将源代码字符串写入一个磁盘上的.java文件：将动态生成的Java源代码保存到磁盘上的一个文件中。
    2. 使用javac命令进行编译：调用外部Java编译器（javac命令）来将.java文件编译为.class文件。
    3. 将编译好的.class文件加载进内存：使用Java的类加载器（通常是ClassLoader的子类）将编译好的.class文件加载到JVM的内存中，以生成相应的Class对象。
    4. 通过Class对象获取并执行相应的方法：一旦有了加载后的Class对象，就可以使用反射等方式调用其中的方法。

    存在问题：

    1. 生成.java和.class文件：在传统的非动态编译方式下，为了编译Java源代码，必须将源代码字符串写入一个.java文件，并通过外部的Java编译器（如javac命令）将其编译成.class文件。这样会在文件系统中生成中间文件，可能会导致磁盘空间的占用和文件管理的复杂性。
    2. 涉及IO操作的耗时：由于涉及将源代码写入文件、编译成字节码文件等IO操作，传统的非动态编译方式可能会导致额外的IO开销，这在大量动态编译的情况下可能会影响性能。

    正因为上述问题，使用传统的非动态编译方式并不总是高效和方便的选择，尤其在需要频繁进行动态编译的情况下，这些IO操作和临时文件管理可能会显著影响应用性能和代码的易读性。

    动态编译技术的引入解决了这些问题。带来了多个优势：

    1. 没有文件生成：由于直接在内存中编译源代码，避免了在磁盘上创建临时.java和.class文件的需求。这有助于保持您的服务器环境的干净，并避免在其中添加不必要的文件。
    2. 没有额外的IO操作：没有文件生成步骤，也就不需要进行读写源代码和字节码文件的IO操作。这使得编译过程更高效和更快速，特别是在频繁或动态生成代码时尤为明显。
    3. 减少复杂性：跳过文件生成和IO操作简化了代码，无需管理临时文件。这使得代码库更加干净，易于维护。

    下边介绍如何使用动态编译API

    Java编译器API

    Java编译器API是指JDK中提供的一组用于在Java应用程序中与Java编译器进行交互的类和接口。这个API允许开发人员在运行时通过编程方式调用Java编译器，从而实现动态编译功能或动态生成Java字节码。

    在JDK中，Java编译器API主要由javax.tools包中的类和接口组成。其中一些重要的类和接口包括：

    1. javax.tools.JavaCompiler: 这是Java编译器的主要接口，它定义了用于编译Java源代码的方法。通过javax.tools.ToolProvider类可以获取JavaCompiler的实例。
    2. javax.tools.ToolProvider: 提供了一些静态方法，用于获取Java编译器和其他工具的实例。
    3. javax.tools.JavaFileObject: 代表Java源文件或类文件的抽象，允许Java编译器在内存中处理Java源代码。
    4. javax.tools.StandardJavaFileManager: 这是一个文件管理器接口，用于管理Java文件对象，例如查找Java源文件或类文件。

    使用Java编译器API，我们可以在Java应用程序中动态地生成Java源代码，然后将其传递给Java编译器进行实时编译。这对于一些特定的用例，如动态生成代码、插件系统或动态加载模块，非常有用。

    这个方法一共有 6 个入参，它们分别是：

    1. out (Writer)：这是编译过程中的输出流，用于捕获编译过程中的标准输出和错误输出。为 null 的话就是 System.err。
    2. fileManager (JavaFileManager)：这是编译过程中的文件管理器，用于处理Java文件对象的查找和管理。您可以通过JavaCompiler.getStandardFileManager方法来获取标准的文件管理器实例。
    3. diagnosticListener (DiagnosticListener<? super JavaFileObject>)：这是诊断监听器，用于收集编译时的错误和警告信息。
    4. options (Iterable)：这是编译选项，包含编译时的一些设置。例如，您可以指定编译输出的目录、编译版本等。
    5. classes (Iterable)：这是一个可选参数，代表类名列表。如果不需要特定的类名，可以传入null。
    6. compilationUnits (Iterable<? extends JavaFileObject>)：这是编译的Java源文件对象列表，表示要编译的源代码。您需要将要编译的Java源文件封装成JavaFileObject对象，并以列表形式提供。

    JavaFileManager fileManager

    JavaFileManager用于处理编译过程中的Java文件对象的查找和管理。它充当编译器和文件系统之间的桥梁，允许编译器与不同的文件系统交互，并能够将源代码和编译结果存储在不同类型的存储介质中，如内存、文件系统或其他自定义存储。

    在动态编译过程中，编译器会将编译生成的字节码存入一个JavaFileObject对象中，而这个JavaFileObject对象是由我们自己编写的。在这个过程中，JavaFileManager的作用非常重要，编译器会调用我们传入的JavaFileManager对象的getJavaFileForOutput方法，这个方法会 new 一个我们写的 TmpJavaFileObject 对象，并把返回给编译器。

    Iterable<? extends JavaFileObject> compilationUnits

    通过将要编译的Java源代码单元传递给compilationUnits参数，我们可以告诉编译器需要编译哪些Java文件。这些Java源代码单元可以是通过JavaFileManager.getJavaFileForInput方法获取的文件对象，也可以是自定义的JavaFileObject对象。

    在动态编译中，通常将要编译的Java源代码表示为一组JavaFileObject对象，并将它们传递给编译器的compilationUnits参数。编译器将根据这些源代码单元执行编译操作，并生成相应的字节码。

    为了使其他操作更方便，我们可以通过继承 SimpleJavaFileObject 来实现我们自己的 JavaFileObject。为了知道我们都需要重写 SimpleJavaFileObject 的哪些方法，我们首先需要看一下 compiler.getTask(…).call() 的执行流程，看看都需要用到什么方法。 compiler.getTask(...).call() 的执行流程如下图所示：

    执行流程说明：

    • 当我们进行动态编译时，首先需要获取源码，这就好比我们拿着一本书，才能开始阅读。编译器会通过JavaFileObject的getCharContent方法，得到源码的内容，就像翻开书页看书一样。

    • 然后，编译器会对得到的源码进行编译，就像把书中的内容理解并转换成我们能听懂的语言。编译器将生成字节码，就像把书中的内容转换成我们能听懂的话语，并且将得到的字节码封装进一个新的JavaFileObject对象。编译器会把生成的字节码存放在一个特殊的地方，类似于将新的内容记录在纸上。但这个地方不是我们自己写的，它是由我们提供的JavaFileManager来管理的，就像我们把内容记录在我们自己的笔记本上。

    • 这时，JavaFileManager登场了！编译器会调用我们传入的JavaFileManager的getJavaFileForOutput方法，就像请求我们打开一个新的笔记本。这个方法会创建一个我们自己写的TmpJavaFileObject对象，并把它返回给编译器，就像我们打开了一本新的笔记本准备记录内容。

    • 然后，编译器会把生成的字节码放进我们提供的TmpJavaFileObject对象中，就像我们把内容记录在我们自己的笔记本上。为了能容纳这些字节码，我们在TmpJavaFileObject中添加了一个“容器”，类似于笔记本上的一页空白纸。为了更好地存储字节码，我们在TmpJavaFileObject中加入了一个ByteArrayOutputStream属性，就像在笔记本上准备了一个专门的空间来写字节码。

    • 编译器会调用openOutputStream()方法，就像请我们打开笔记本上的一页空白纸，并将字节码写在上面。编译器将把生成的字节码写入到我们提供的“纸上”，也就是ByteArrayOutputStream容器中。最后，为了得到最终的结果，我们需要将这些字节码从“纸上”取下来，就像我们需要把内容从笔记本上拿出来查看一样。我们通过在TmpJavaFileObject中添加一个getCompiledBytes()方法，将ByteArrayOutputStream中的字节码转换成byte[]字节数组，并将它返回。

    这样，我们就成功地进行了动态编译，并获得了编译生成的字节码。这种灵活的动态编译技术，就像是我们在阅读和记录新的内容时，有了自己的特定方式，让整个过程更加顺利和高效。
