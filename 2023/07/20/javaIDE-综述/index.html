    java在线IDE

    发表于 2023-07-20 分类于 研究生 ， Java Spring

    本项目使用 Spring Boot 实现了一个在线的 Java IDE，允许客户端提交 Java 代码的 main 方法，并在远程服务器上运行。服务器将程序的标准输出内容和运行时异常信息返回给客户端。同时，项目还对客户端提交的程序执行时间进行了限制。

    在项目中，涉及到的框架知识相对简单，主要集中在 Java 基础方面。例如，：Java 程序编译和运行的过程、Java 类加载机制、Java 类文件结构、Java 反射等。除此之外，还涉及到了一个简单的并发问题：如何将一个非线程安全的类变为一个线程安全的类。

    涉及技术

    • Java 动态编译

    • Java 类文件的结构

    • Java 类加载器 & Java 类的热替换

    • Java 反射

    • 如何将一个类变为线程安全类

    项目模块

    • 实现编译模块： 使用动态编译技术，可将客户端发来的源代码字符串直接编译为字节数组；
    • 实现运行模块： 自定义类加载器实现类的加载 & 热替换，通过反射实现 main 方法的运行；
    • 实现自定义System类： 根据 Java 类文件结构修改类的字节码，可将客户端程序对 System 的调用替换为对 System的替代类 HackSystem 的调用；
    • 解决多用户同时发送执行代码请求时的并发问题： 通过 ThreadLoacl 实现线程封闭，为每个请求创建一个输出流存储标准输出及标准错误结果；

    项目实现流程

    Java 程序运行全过程

    首先我们看一下java程序从源码到最终运行的整个过程

    1. 编写Java源代码：首先，开发人员编写Java源代码，使用.java作为文件扩展名。
    2. 编译Java源代码：使用Java编译器（javac）对Java源代码进行编译。编译的过程会将源代码转换为字节码文件（.class文件），字节码是一种中间代码，它不是直接在操作系统上运行，而是在Java虚拟机（JVM）上执行。
    3. 类加载：在Java程序运行时，JVM会负责加载字节码文件到内存中。类加载是Java虚拟机执行的一个过程，它包括三个主要步骤：加载、连接、初始化。加载是指将类的字节码文件加载到JVM中。连接包括验证、准备和解析步骤，其中验证确保字节码的正确性和安全性，准备负责为类的静态变量分配内存并设置默认初始值，解析将符号引用转换为直接引用。初始化阶段是类加载的最后一步，在此阶段，JVM会执行类的初始化代码，为静态变量赋予正确的初始值。
    4. 执行Java程序：类加载完成后，JVM会执行Java程序的main方法作为程序的入口点。在main方法中，Java程序开始执行，按照预定义的逻辑进行运行。

    需要注意的是，Java程序的执行是在Java虚拟机上进行的，这意味着Java是一种解释性语言。JVM会逐行解释字节码并将其转换为本地机器码，然后由底层操作系统执行。

    总的来说：

    • 源文件由编译器编译成字节码；
    • 字节码由 Java 虚拟机解释运行。

    编译

    在Java编译过程中，编译器会根据源代码中的依赖关系逐个编译类，并在需要时引用它们。如果在编译过程中找不到所依赖的类的.class文件或.java源文件，编译器将报告“cannot find symbol”的错误。

    这种依赖关系在Java中被称为类的依赖关系图。编译器会根据依赖关系图来确定编译顺序，确保所有的依赖类在需要时都已经编译好并可以访问。

    例如，假设我们有两个类：ClassA和ClassB，其中ClassA依赖于ClassB。如果我们先编译ClassA.java文件，编译器会检查ClassA的源代码，发现它依赖于ClassB，于是会先编译ClassB.java（如果ClassB尚未编译），然后再编译ClassA.java，确保所有依赖都被满足。

    如果在编译ClassA.java时，编译器找不到ClassB的.class文件或.java源文件，它就会报告“cannot find symbol”的错误，因为它无法解析ClassB类。

    运行

    Java类的运行可以分为两个主要过程：类的加载和类的执行。

    1. 类的加载： 在应用程序运行时，JVM进程会从类路径（classpath）中找到对应的二进制文件（.class文件），并将该类的信息加载到运行时数据区的方法区内。这个过程被称为类的加载。

    类的加载由Java的类加载器（ClassLoader）来完成。类加载器负责将类的字节码文件加载为Java虚拟机中的Class对象，这个Class对象包含了类的各种信息，如类的结构、字段、方法等。加载后的Class对象存放在方法区中，供JVM在运行时使用。

    Java类加载器采用了双亲委派模型，它会优先使用父类加载器尝试加载类。如果父类加载器无法加载该类，再由当前类加载器进行加载。这样确保了类的加载过程的一致性和安全性。

    1. 类的执行： 一旦类加载完成，JVM会找到类中的主函数入口（通常是public static void main(String[] args)方法），并开始执行main函数。类的执行即程序的运行阶段，JVM按照main方法中定义的逻辑开始执行Java程序。

    在本项目中，主要使用了反射来实现类的执行过程。通过反射，我们可以在运行时获取并调用类的方法，这允许程序动态地加载和执行客户端提交的Java代码。

    反射是Java的一项强大功能，它允许程序在运行时动态地获取类的信息和调用类的方法，是实现Java IDE中在线运行代码的重要技术之一。
